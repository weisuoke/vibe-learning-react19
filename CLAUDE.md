# 原子化知识点生成规范

> 本文档定义了为 React19 源码学习项目生成原子化知识点文档的标准和要求

---

## 📋 文档概述

**项目目标：** 为 React19 源码学习构建完整的原子化知识体系

**核心理念：**
- **原子化**：每个知识点独立完整，可独立学习
- **实战导向**：所有知识点都要联系 React 源码理解和实际应用
- **初学者友好**：假设零基础，用简单语言和丰富类比
- **速成高效**：抓住20%核心解决80%问题

---

## 🎯 生成流程

### 第一步：确认输入信息

在开始生成前，确认以下信息：

1. **知识点名称**：从 `atom/[大类]/k.md` 中获取
2. **大类目录**：如 `01_JavaScript核心`、`02_数据结构与算法`、`03_Fiber架构` 等
3. **目标受众**：初学者（默认）或进阶学习者
4. **类比领域**：生活场景（默认）或其他领域
5. **文件位置**：`atom/[大类]/[编号]_[知识点名称].md`

### 第二步：读取模板接口

**模板文件：** `prompt/atom_knowledge.md`

**10个必需维度：**
1. 【30字核心】
2. 【第一性原理】
3. 【3个核心概念】
4. 【最小可用】
5. 【1个类比】
6. 【反直觉点】
7. 【实战代码】
8. 【面试必问】
9. 【化骨绵掌】
10. 【一句话总结】

### 第三步：按规范生成内容

参考下方的详细规范生成每个维度的内容。

### 第四步：质量检查

使用文档末尾的检查清单验证质量。

## 📐 10个维度的详细规范

### 1. 【30字核心】

**要求：**
- 用**一句话**说清知识点的本质
- 字数控制在**30字左右**（可略有浮动）
- 必须包含：定义 + 作用/价值
- 使用**加粗**突出显示

**模板：**
```markdown
**[知识点] 是 [定义]，[作用/在 React 源码中的价值]。**
```

**示例：**
```markdown
**Fiber节点是React内部的虚拟DOM节点，通过child/sibling/return三指针构建链表结构，是React实现可中断渲染的基础。**
```

---

### 2. 【第一性原理】

**要求：**
- 回到**最基础的真理**思考知识点
- 包含：
  1. 最基础的定义（不可再分）
  2. 为什么需要这个概念（从根本问题出发）
  3. 核心价值（解决了什么问题）
  4. 从第一性原理推导应用
  5. 一句话总结

**结构模板：**
```markdown
### 什么是第一性原理？

**第一性原理**：回到事物最基本的真理，从源头思考问题

### [知识点]的第一性原理 🎯

#### 1. 最基础的定义

**[知识点] = [最基本的定义]**

仅此而已！没有更基础的了。

#### 2. 为什么需要[知识点]？

**核心问题：[根本性问题]**

#### 3. [知识点]的三层价值

##### 价值1：[价值名]
[解释 + 示例]

##### 价值2：[价值名]
...

#### 4. 从第一性原理推导 React 实现

**推理链：**
```
1. [前提1]
   ↓
2. [推导2]
   ↓
3. [推导3]
   ↓
...
   ↓
N. [最终应用/结论]
```

#### 5. 一句话总结第一性原理

**[知识点]是[本质]，[核心价值]。**
```

---

### 3. 【3个核心概念】

**要求：**
- 提炼**3个最核心的概念**
- 每个概念包含：
  - 概念名称（加粗）
  - 一句话定义
  - 详细解释
  - 代码示例
  - 可视化（如适用）
  - 在 React 源码/开发中的应用

**结构模板：**
```markdown
### 核心概念1：[概念名] 🔢

**[一句话定义]**

```javascript
// 代码示例
```

**[详细解释]**

**在 React 源码/开发中的应用：**
[实际应用说明]
```

**选择标准：**
- 最基础、最重要的概念
- 后续学习的依赖项
- 实际应用中最常用的

---

### 4. 【最小可用】

**要求：**
- 提炼出**20%的核心知识**解决**80%的问题**
- 列出3-5个必须掌握的要点
- 每个要点包含：
  - 概念说明
  - 代码示例（如适用）
  - 实际应用场景

**结构模板：**
```markdown
掌握以下内容，就能理解 React 源码核心：

### 3.1 [核心知识点1]
[简要说明]
```javascript
// 代码示例
```

### 3.2 [核心知识点2]
...

**这些知识足以：**
- [能做什么1]
- [能做什么2]
- [为后续学习打基础]
```

**要点：**
- 聚焦**最常用**的知识
- 避免边缘案例和高级用法
- 与 React 源码理解直接相关

---

### 5. 【1个类比】

**要求：**
- 使用**生活场景**类比（默认）
- 如果用户指定其他领域，使用指定领域
- 提供**3-5个具体类比**
- 每个类比包含：
  - React 源码概念
  - 生活场景对应概念
  - 详细解释相似性

**结构模板：**
```markdown
### 类比1：[React概念] = [生活场景] 🎨

[解释相似性]

**举例：**
[具体的生活例子]

```javascript
// JavaScript代码示例
const example = {
  // 对应React源码的某个特性
};
```

---

### 类比2：...
```

**推荐生活场景类比领域：**
- 家庭关系：家谱树、家庭成员
- 日常管理：图书管理、排队系统
- 生活场景：装修、做饭、找不同游戏
- 交通系统：高速公路、车道分配
- 时间管理：待办清单、闹钟设置

**类比总结表：**
提供一个表格总结所有类比。

---

### 6. 【反直觉点】

**要求：**
- 列出**3个最常见的误区**
- 每个误区包含：
  - ❌ 错误观点（简洁陈述）
  - **为什么错**（正确解释）
  - **为什么人们容易这样错**（心理/认知原因）
  - **正确理解**（配代码示例或类比）

**结构模板：**
```markdown
### 误区1：[错误观点] ❌

**为什么错？**
- [正确解释]
- [关键区别]

**为什么人们容易这样错？**
[心理/认知原因，联系日常经验]

**正确理解：**
```javascript
// 代码示例说明正确理解
```
```

**示例主题：**
- 概念混淆（如"Fiber = 虚拟DOM"）
- 范围局限（如"Hooks只能用在函数组件"）
- 时机误解（如"useEffect在渲染前执行"）

---

### 7. 【实战代码】

**要求：**
- 提供**一个完整可运行**的代码示例
- 代码必须：
  - ✅ 可以直接复制运行（或在注释中说明如何运行）
  - ✅ 包含详细注释
  - ✅ 展示实际应用场景
  - ✅ 输出结果清晰
- **混合代码风格**：
  - 基础概念用**简化的JavaScript实现**（便于理解原理）
  - 进阶内容引用**真实React源码片段**（贴近实际）
- 包含预期输出示例

**结构模板：**
```markdown
**基础实现（简化版）：**
```javascript
// ===== 1. [场景1：基础概念演示] =====
console.log("=== [场景描述] ===");

// [操作说明]
code_here

// ===== 2. [场景2：核心功能] =====
...
```

**运行输出示例：**
```
[预期的控制台输出]
```

---

**进阶：React源码实现**
```javascript
// React源码片段（packages/xxx/src/xxx.js）
// 注释说明关键逻辑
```
```

**代码规范：**
- 变量命名清晰（如 `fiberNode` 而非 `f`）
- 每个代码块前加说明注释
- 使用 `console.log()` 输出关键结果
- 复杂逻辑拆分成小步骤
- 标注代码来源（简化实现 vs 真实源码）

---

### 8. 【面试必问】

**要求：**
- 设计1-2个高频面试问题
- 提供**对比答案**：
  - ❌ 普通回答（不出彩）
  - ✅ 出彩回答（推荐）
- 出彩答案要包含：
  1. 多层次解释（原理/实现/应用）
  2. 具体例子
  3. 与 React 源码设计的联系
  4. 展示深度思考

**结构模板：**
```markdown
### 问题："[面试问题]"

**普通回答（❌ 不出彩）：**
"[简单/肤浅的回答]"

**出彩回答（✅ 推荐）：**

> **[知识点] 有三层含义：**
>
> 1. **[层面1]**：[解释]
>
> 2. **[层面2]**：[解释]
>
> 3. **[层面3]**：[解释]
>
> **与[相关概念]的区别**：[对比说明]
>
> **在实际工作中的应用**：[举例说明在 React 项目/性能优化/架构设计中的应用]

**为什么这个回答出彩？**
1. ✅ [优点1]
2. ✅ [优点2]
3. ✅ [优点3]
```

## ✍️ 内容撰写原则

### 1. 初学者友好原则

**语言：**
- ✅ 使用简单、日常的语言
- ✅ 避免未定义的专业术语
- ✅ 必须使用的术语要先解释
- ❌ 避免复杂的数学符号（必要时配文字说明）

**结构：**
- 从简单到复杂
- 从具体到抽象
- 从已知到未知

**解释方式：**
- 多用类比和比喻
- 提供可视化（文字图表）
- 举具体例子

### 2. React 源码相关性

**每个部分都要联系实际应用：**
- ✅ 这个知识在 React 源码中如何体现？
- ✅ 为什么 React 需要这样设计？
- ✅ 实际场景举例（组件渲染、状态管理、性能优化）

**重点强调：**
- Fiber架构的设计思想
- 调度和协调机制
- Hooks的实现原理
- React 19的新特性

### 3. 代码规范

**JavaScript/TypeScript代码：**
```javascript
// 1. 导入语句清晰
import React from 'react';

// 2. 变量命名有意义
const fiberNode = {  // ✅
  type: 'div',
  child: null
};
const f = { };       // ❌

// 3. 添加注释说明
// 创建一个Fiber节点表示div元素
const divFiber = {
  type: 'div',
  child: null,
  sibling: null
};

// 4. 输出关键结果
console.log(`Fiber类型: ${divFiber.type}`);

// 5. 分段组织（用注释分隔）
// ===== 1. 创建Fiber节点 =====
...

// ===== 2. 建立Fiber关系 =====
...
```

**混合代码风格：**
- 基础概念：用简化的JavaScript实现
- 进阶内容：引用React真实源码片段
- 明确标注代码来源

### 4. Markdown格式规范

**标题层级：**
```markdown
# 主标题（知识点名称）
## 一级标题（10个维度）
### 二级标题（维度内的子章节）
#### 三级标题（详细说明）
```

**强调标记：**
- **加粗**：重要概念、关键词
- `代码`：代码片段、变量名、文件名
- > 引用：出彩回答、核心观点

**列表：**
- 无序列表：要点说明
- 有序列表：步骤说明
- 检查清单：`- [ ]` / `- [x]`

**代码块：**
````markdown
```javascript
// JavaScript代码
```

```typescript
// TypeScript代码
```

```css
/* CSS代码 */
```
````

**表格：**
用于对比、总结、速查

**emoji使用：**
适度使用emoji增加可读性，但不要过度。

### 5. 类比策略

**生活场景类比的推荐方向：**

| React源码概念 | 生活场景类比 |
|--------------|-------------|
| Fiber双向链表 | 家谱树（父母、孩子、兄弟姐妹关系） |
| Reconciler协调器 | 图书管理员整理书架（比对、移动、删除） |
| Scheduler调度器 | 医院分诊台（按优先级排队） |
| 双缓冲树 | 装修：住在旧房子，装修新房子，完工后搬家 |
| Diff算法 | 找不同游戏（快速定位差异） |
| 时间切片 | 做饭时穿插洗碗（避免一直切菜手累） |
| Lane优先级 | 高速公路车道（快车道、慢车道） |
| Hooks链表 | 待办事项清单（按顺序执行） |
| Effect调度 | 闹钟设置（在特定时机触发） |
| 批量更新 | 打包快递（攒够了一起发） |

**类比要求：**
- 准确（不能误导）
- 易懂（日常生活中熟悉）
- 具体（有详细解释）

## 🚀 快速启动模板
